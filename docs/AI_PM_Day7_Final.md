# Day 7 结课作业：近代史纲要 · AI 刷题台

## 1. 产品基本信息

**产品名称**：近代史纲要 · AI 刷题台  
**一句话介绍**：给正在学《中国近现代史纲要》的大学生一个「学号登录、随时刷题、AI 逐题解析、错题智能画像」的在线复习工具，让考前复习从死记硬背变成可理解、可追踪、有反馈。  
**目标用户**：高校本科生，正在修读或备考《中国近现代史纲要》公共必修课。  
**核心使用场景**：
- 考前 1-3 周：利用碎片时间刷单元题或全书题；
- 课堂/自习后：对课上没听懂的知识点，通过 AI 详细解析逐项搞懂每个选项；
- 考前冲刺：进入错题画像，看薄弱章节和知识点，针对性重做错题。

## 2. 需求来源

这个产品的需求来自三个观察：

1. **题库大，复习无从入手**：《中国近现代史纲要》是公共必修课，考试前学生需要面对数百道选择题，很多人不知道从何刷起。
2. **同一知识点反复错**：时间、条约、人物、地点等史实容易记混，传统文字解析往往只说「选 A」，不讲「为什么 B/C/D 错」。
3. **缺少个人化追踪**：传统纸质或表格题库刷完就忘，学生无法定位自己真正的薄弱点。

需求最初来自自己和身边同学的考前复习痛点，以及学校课程辅助工具长期缺失的现实。

## 3. 产品方案设计

### 产品定位
一个轻量级 Web 刷题平台，核心不是「替代老师」，而是成为学生课后自主复习的 AI 助教：点击即答、即时反馈、AI 讲解、数据追踪。

### 核心功能

| 功能 | 说明 |
|------|------|
| 学号登录 | 输入 8 位学号即可使用，答题记录自动关联到个人 |
| 多种练习模式 | 单元测试（按章）、全书测试、错题本 |
| 点击即答 | 选择选项后自动提交并显示结果 |
| AI 解析 | 每题支持 AI 提示 / 详细解析 / 错题复盘；详细解析逐项分析 A/B/C/D |
| AI 复习助手 | 右侧抽屉，可结合当前章节/题目自由提问 |
| 错题画像 | 章节正确率、薄弱知识点 TOP、高频错题统计 |
| 本章资料 | 知识点卡片、事件时间轴、Leaflet 地理地图、书后配套分析题 |
| 全书总览 | 跨章节知识点、总时间轴、总事件地图 |

### 核心交互流程

1. 学号登录 → 系统自动关联历史答题记录；
2. 选择模式 → 单元/全书/错题，设置题量与乱序；
3. 点击答题 → 选择选项自动提交，即时反馈对错；
4. 查看解析 → 显示正确答案、内置解析、AI 详细解析入口；
5. AI 逐项解析 → 对 A/B/C/D 分别说明史实背景与对错原因；
6. 错题画像 → 查看章节正确率、薄弱知识点、高频错题；
7. 返回复习 → 从画像返回刷题页，自动回到原题；
8. 持续迭代 → 每次作答都保存，错题本与薄弱知识点动态更新。

## 4. AI 工作流程

### 输入
- 题目文本、选项 A/B/C/D、正确答案、用户答案、章节 key、解析模式（hint / explain / review）

### 处理
- 服务端组装 System Prompt：包含章节知识库、时间轴、输出格式要求；
- 拼接 User Prompt：题目 + 用户答案 + 模式；
- 调用 `mimo-v2.5-pro-ultraspeed`，流式返回；
- 不设置 `max_tokens`，避免 reasoning_content 占用额度导致正文为空；
- 保留 30 秒 timeout 和 Zod 输入校验。

### 输出
- Markdown 格式自然语言解析，SSE JSON chunk 流式返回到前端；
- 前端用 react-markdown + remark-gfm 渲染。

### 用户确认环节
- AI 解析是可选增强，内置解析始终可用；
- 用户可主动关闭 AI 抽屉，自主控制 AI 介入深度；
- 系统判断对错不依赖 AI，避免模型幻觉改答案。

## 6. 预期收益

### 用户价值
- 让考前复习更高效：从「盲目刷题」转向「精准补弱」；
- 让历史学习更可理解：AI 逐项解析讲清每个选项背后的史实；
- 让个人进步可追踪：错题画像帮助学生定位真正薄弱的知识点。

### 关键指标目标

| 指标 | 目标 | 说明 |
|------|------|------|
| 登录完成率 | > 90% | 学号登录无门槛 |
| AI 解析使用率 | > 40% | 答错题后愿意点开 AI 解析 |
| 错题画像访问率 | > 30% | 刷题后主动查看个人薄弱点 |
| 从画像返回刷题率 | > 25% | 画像能带动二次练习 |
| 周留存率 | > 30% | 学生愿意持续使用 |

### 潜在商业价值
- 免费核心功能吸引学生；
- 付费学期卡/年卡：无限 AI 解析、错题导出、考前模拟卷；
- 班级/学校企业版：教师端查看班级薄弱知识点；
- 通过班级渗透和内容种草实现低成本获客。

## 7. 项目复盘

### 收获
1. 完整跑通了从产品想法、核心交互流程、AI 接入、数据库设计到部署上线的闭环；
2. 理解了 AI 产品中人机边界的重要性：AI 负责解释，系统负责判定，用户负责决策；
3. 学会了如何处理 AI 模型的实际工程问题（如 `reasoning_content` 占用 token、SSE 流式返回、超时兜底等）；
4. 积累了用 AI Coding 工具快速迭代全栈应用的经验。

### 遇到的问题
1. **AI 解析偶尔为空**：原因是模型 `reasoning_content` 占用 token 额度，解决方式是移除 `max_tokens` 限制；
2. **Standalone 构建后 CSS 404**：原因是 `.next/static` 未同步到 standalone 目录，解决方式是在部署脚本中加入复制步骤；
3. **地图 marker 偏移**：原因是高德使用 GCJ-02 坐标，解决方式是 WGS84 → GCJ-02 转换；
4. **AI 文案有 AI 味**：引入 `humanize-zh` skill 对文案进行去痕润色。

### 当前不足
1. 真实用户数据还不够，主要依赖自己和身边同学测试；
2. AI 解析偶尔仍有机器感，需要持续优化 prompt 和文案；
3. 班级渗透和教师端尚未启动；
4. 商业化闭环还未验证。

### 下一步迭代计划
1. **提升 AI 解析质量**：使用 `humanize-zh` 润色解析示例，减少 AI 味；
2. **验证班级渗透**：联系学习委员/助教，在考前分享到班级群，收集真实数据；
3. **接入教师端**：提供班级错题 TOP、薄弱知识点报告，从学生工具延伸到教学辅助工具；
4. **小规模付费实验**：期末考前测试「考前冲刺包」「错题导出 PDF」的付费意愿；
5. **AI 生成复习计划**：根据错题画像自动生成个性化复习建议。

**结课总结**：通过这次共学营，把一个真实痛点做成了可上线、可体验、可迭代的产品原型。最重要的收获不是功能做完了多少，而是建立了「从问题出发、用数据验证、让人机各安其位」的 AI 产品思维。
